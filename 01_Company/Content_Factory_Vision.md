# Content Factory 願景（2026-07-06 定案）

> 這份文件記錄的不是單一專案的待辦，是公司整體的營運模式願景——從「經營一個商城網站」轉變成「經營一間內容工廠（Content Factory）」，商城（ST）只是眾多出口之一。

## 核心洞察

原本規劃「商品型態規劃」（每個Content建立時勾選預計製作的電子書/課程/有聲書/模板/Podcast/AI工具等）時，是用「ST商城要改版」的角度在看。但這件事真正的目的不是商城本身，而是要幫**整個內容資產庫**建立一份完整、一次規劃到位的「生產藍圖」，讓之後不管是CC、龍蝦（AI員工代稱）、或其他AI，都能直接照著藍圖生產，不用每次重新開會討論方向。

**這句話是核心**：現有商品（目前約80~100個Content：AI書院/心理學書院/股市書院/腦中的宇宙/未來的寵物書院等）**規劃完成才繼續新產品規劃，多步合一進行**——不是「一本做完才想下一本」，而是先把現有全部Content一次規劃完整、形成完整商品矩陣，之後只要有空就依矩陣生產，不用再重新思考方向。

## Content Asset Matrix（內容資產矩陣）

每個Content不再只是單一格式，而是有一張完整的發展藍圖：

```
AI 入門
  ✓ 電子書（已上市）
  ✓ Readmoo
  ✓ Pubu
  ✓ 線上閱讀
  □ 課程
  □ 有聲書
  □ Podcast
  □ AI工具
  □ 模板
  □ 畫冊
  ...
```

每個格式可以再加優先度評分，形成「生產排程」，例如：

```
Content A：課程★★★★★／Podcast★★／AI工具★★★★★
Content B：課程★★／模板★★★★★／有聲書★★★★
```

有了這份矩陣，任何AI（CC、龍蝦、其他AI）都能直接查「下一個最值得做的是什麼」，不用每次重新討論。

**技術落地備註**：這跟同一天（2026-07-06）ST商城討論的`Content.productStatus`（Phase1的domain model）+`ProductType`主檔是同一套機制的自然延伸——`productStatus`本來就是陣列式`[{productType, status}]`設計，天生就能承載這份矩陣；只是目前還缺「後台勾選UI」跟「全部Content一次盤點填滿」這兩步（見ST repo memory `project_commerce_phase5_scope_20260706.md`的Prerequisites）。

## 六條生產線

不再只用「商城專案」的角度規劃，而是拆成六條並行的生產線：

| 生產線 | 內容 |
|---|---|
| ① 內容生產線 | 新書規劃、電子書撰寫、課程製作、AI工具開發、模板製作、有聲書錄製 |
| ② 商品化生產線 | 商品型態規劃、上架ST/Readmoo/Pubu/線上閱讀、定價與促銷 |
| ③ 平台經營線 | Facebook／Instagram／Threads／X／YouTube／LINE官方帳號，每天都有內容要發 |
| ④ 短影音生產線 | 一篇電子書拆成Shorts/Reels/TikTok/YouTube Shorts，一篇內容可拆十幾支短影音 |
| ⑤ SEO／網站流量 | 部落格文章、Landing Page、關鍵字優化、內部連結、網站更新 |
| ⑥ 營運分析 | 哪本書賣最好、哪個平台流量最高、哪種短影音效果最好、哪個主題最值得延伸 |

## 「龍蝦」= AI員工角色分工

「龍蝦」是內部對AI工作代理人（agent）的稱呼。之前理解龍蝦只是「幫CC看程式、跑測試」，這次重新定義：龍蝦成熟後，每隻都能有自己的專職工作，不會沒事做——因為專案本身就是一個持續運作的內容生態系：

| 龍蝦 | 職責 |
|---|---|
| 📚 內容龍蝦 | 整理內容、建立商品規劃 |
| 🎬 影音龍蝦 | 把文章拆成短影音腳本 |
| 📱 社群龍蝦 | 安排各平台發文、追蹤成效 |
| 🛒 商城龍蝦 | 檢查商品狀態、上架流程 |
| 📊 數據龍蝦 | 分析銷售與流量，提出建議 |
| 💻 開發龍蝦 | 協助CC處理程式、自動化與測試 |

## 妹的角色 vs AI團隊的角色

- **妹**：決定做什麼主題、決定品質是否過關、決定優先順序（編輯/出版判斷，見既有分工`feedback_c_suggestions`延伸的CEO/CTO分工原則）
- **AI團隊**：生產、包裝、上架、宣傳、分析、持續優化

目的：把妹的時間留給最有價值的決策，而不是每天重複做執行工作。

## 這代表什麼

「有的沒的小舖不是一個網站，是一間AI公司。」商品型態規劃不是為了商城改版，是為了替整個AI團隊建立一份可以長期執行的「工作清單」——當每個Content都完成商品型態、平台拓展與宣傳規劃後，不論是Claude Code、未來的龍蝦、還是其他AI，都能直接依照規劃持續產出，不用每次重新討論方向。這樣才能做到「多步合一，同步推進」：**之後是持續生產，而不是持續規劃。**
