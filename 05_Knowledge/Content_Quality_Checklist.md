# 給C：內容品質檢查表

> 2026-07-12從Desktop散檔（`有的沒的小舖\驚喜樂世界\驚喜學院\給C-內容品質檢查表.md`）搬進AIOS集中管理，原路徑只留指標。跨系列通用標準，不限AI書院，任何新系列開工前直接丟給C這份文件。

【給C：動筆前完整讀一次，這不是形式流程，是防止內容變成「湊字數」的實質檢查點】

CC（技術執行）跟GPT（編輯視角）各自獨立想過這個問題，這份是合併兩邊後的最終版。每一題都要誠實回答，答不出來就代表這個段落/這堂課還沒準備好。

---

## 核心原則

讀者付249元買的不是「資訊」，是「別人已經幫他想清楚、組織好、做出判斷的思考路徑」。

單次問AI通常會得到「A有優點也有缺點，B也是，要看情況」這種安全但空洞的答案。你寫的每一堂課，必須做到讀者自己問AI一次得不到的東西——一個具體的判斷、立場，或是機制性的解釋加上具體到位的例子。

如果整堂課讀完找不到任何一句「這是作者的判斷、可能有人不同意」的話，就是不及格。

---

## 2026-07-12補充：免費/入門層級也適用同一把尺

同一份內容可以用多種面貌呈現（電子書／遊戲片段／有聲書／短片／進階課程），吸引不同類型的人，但不管哪種面貌，**免費或入門層級都要是「真的能學到知識」，不能是騙3歲小孩的空洞內容**。這不是加碼，是這份檢查表原本的標準本來就該適用到免費內容上，不要因為「反正是免費的」就放寬。

## 2026-07-12補充：自我檢查要「留下痕跡」，不能只是想過

CC現在會在`pre-epub-check.mjs`裡機械檢查有沒有留下這兩個標記，沒留=擋下build，不能進到下一步。所以第0題跟交稿前紅燈訊號不能只是在腦中想過，要實際寫進MD裡，格式如下（HTML註解，讀者看不到，只是留給CC的機器檢查用）：

**每一堂（`# 第X堂`）結尾，寫一行：**
```
<!-- SELFCHECK：這堂的具體例子是：[貼上這堂實際用的具體例子，一句話] -->
```

**全書任何位置（建議放在最後一堂或全書結尾）寫一行：**
```
<!-- SELFCHECK：讀者為什麼不能只問ChatGPT：[一句話講差異化在哪] -->
```

填不出來就代表這堂/這本還沒準備好，不要留空硬填，回頭重寫內容。

## 開工前：第0題（比原本3題更硬的一題）

在回答「這冊有幾堂課／每堂核心概念是什麼／讀者讀完具體多懂了什麼／跟免費資源差異化在哪」這4題之後，再加這一題：

> **這堂課有沒有一句話，是讀者自己問AI一次大概率不會得到、需要我做了額外判斷或整理才寫得出來的？請直接寫出那句話。**

不是「保證會有差異化」，是**現在就把那句話寫出來**。
- 寫得出來 → 可以開始動筆
- 寫不出來 → 這堂課題目或角度需要重新想，不要先開工再說

## 寫作中：每寫完一小節，立刻自問（不是交稿後才檢查）

**1. 可替換測試**
把這段文字的主題詞換掉（例如「AI偏見」換成「投資風險」），這段話還成不成立？
→ 還成立 = 填充句／萬用廢話，刪掉重寫。
→ 換了主題就不通 = 這段話跟主題真的相關，保留。
（這個測試同時涵蓋「這段拿掉會不會少學到東西」——通不過可替換測試的句子，拿掉也不會少學到東西）

**2. 具體到可能講錯的程度**
這段有沒有具體到「可能被抓包講錯」的內容（數字、因果關係、具體情境）？還是永遠正確、怎麼講都不會錯的安全句？
→ 只有安全句 = 空洞，需要補上具體判斷或案例。

**3. 角色是不是在做事**
例子裡出現的角色（阿明/小美/阿偉/阿志），是在做一件具體的事、講一句具體的話、遇到一個具體情境？還是只是名字被提到、情境是空的？
→ 名字被提到但情境空 = 例子不合格，要重寫到「具體到有名字、有數字、有因果」的程度。

**4. 結構完整**
問題/情境 → 分析/原理 → 例子 → 一句話帶得走的結論，四環節都要在（詳見下方「一堂課的完整定義」有第五環節）。

**5. 湊字歸因自問**
「我現在寫的這句話，是因為這堂課需要它，還是因為我需要湊到目標字數/堂數？」
→ 誠實回答為湊字 = 刪掉，不要留著填篇幅。
→ **這是六項裡唯一一個能在生成當下打斷慣性的問題，其他項目都是寫完之後才驗。寫的時候持續問這一句，比事後逐條檢查更有效。**

## 一堂課的「完整」定義（五個環節，缺一不算完整）

1. 提出一個具體問題（不是「今天來聊聊AI偏見」，而是「為什麼同一個模型對措辭幾乎相同的問題會給出不同答案」）
2. 給出機制性解釋（為什麼，不只是是什麼）
3. 用一個具體到有名字、有數字、有因果的例子把機制走一遍
4. 指出這個機制的限制或例外（沒有例外的解釋通常代表沒想清楚）
5. 給讀者一個可以帶走、可操作的判斷準則

**字數是雙向警訊，不是單一目標：** 走完五環節，中文大約1500-2500字是合理範圍。低於1000字通常走不完論證閉環；**超過3000字通常是在同一論點繞圈子，也是質量問題**（不是越長越好）。

## 交稿前：紅燈訊號自問（填不出來就代表有問題）

**1.** 「這堂課裡我做出的一個具體判斷/立場是什麼？」
→ 填不出來 = 整堂課在迴避立場，是空洞的安全牌。

**2.** 「如果只能留一段，我會留哪一段？為什麼其他段落沒有它重要？」
→ 覺得每段都差不多重要 = 內容是平鋪直敘的資訊堆疊，沒有主次判斷，這正是「湊」的特徵。

**3.** 「這堂課有沒有可能被讀者反駁的地方？」
→ 完全無法被反駁 = 完全沒有價值判斷。

**4.** 「讀完這節，我自己能不能一句話講出重點？」
→ 講不出來（只覺得「看了但不知道重點」）= 這節本身也沒有明確重點。

## 合格 vs 不合格對照範例

**不合格範例1（填充句/安全句）：**
> 「AI的判斷可能會受到很多因素影響，我們在使用時要多加注意，才能做出更好的決策。」
（換成任何主題都成立，沒有具體判斷，沒有可能被反駁的內容）

**合格範例1：**
> 「阿明問AI『這支股票該不該買』跟『這支股票值得長期投資嗎』，得到完全相反的建議。原因不是AI在說謊，而是『買不買』觸發的是短期價格判斷框架，『值不值得長期投資』觸發的是基本面框架——同一個模型裡其實藏著好幾套互斥的推理路徑，措辭決定了你叫醒哪一套。」
（具體情境、具體因果、可能被反駁——有人可以說「不對，這是因為訓練資料不同」，這才代表這句話有實質內容）

**不合格範例2（角色被提名字但沒在做事）：**
> 「就像小美之前提到的，AI在職場上的應用越來越廣泛。」

**合格範例2（角色真的在做一件具體的事）：**
> 「阿志把AI當『他的學生和外部記憶體』。他不斷把自己的業界判斷輸入系統，讓AI學習他的邏輯；同時，他讓AI幫他把十二年的工作經驗整理成文字。」

**不合格範例3（只有問題+結論，中間空心）：**
> 「AI會不會取代你的工作？答案是：會，也不會，取決於你怎麼用它。」
（問題丟出來直接跳結論，沒有分析過程、沒有例子、沒有限制或例外，讀者無法判斷這個結論怎麼來的）

**合格範例3（五環節都在）：**
> 情境：「小林跟小陳同樣面對公司導入AI客服系統……」
> 分析：「AI最能接手的是有標準答案、重複性高的任務……」
> 例子：「小林選擇抵制新工具，小陳則主動學習怎麼跟AI協作，半年後……」
> 限制/例外：「但如果客服工作本身高度依賴臨場情緒判斷，這個轉型邏輯就不成立。」
> 結論：「同樣的技術衝擊，主動調整和被動等待的結果差很多，但前提是這份工作真的有『可被AI接手的重複性核心』。」

## 交稿流程

1. C 寫完後，自己對照上面所有測試逐節檢查一次，標記出沒通過的段落並自己修掉或補齊
2. 妹/GPT 做第二輪抽查：邏輯連貫性、跨系列會不會重複講同一個案例、隨機抽幾節做「讀者一句話測試」模擬
3. 兩輪都過，才進入CC的技術上架流程（epub建置、圖片、格式檢查）

## 給C的一句話提醒

寫的當下持續問自己：這句話是「因為這堂課需要它」，還是「因為我需要填滿篇幅」？分不清楚的時候，答案通常是後者——那就刪掉。
